# 完整日志类型分类

## 概述

本文档基于 `e:\\codex\\道具日志调查\\日志类型说明.md` 文件，提供完整的游戏日志类型分类和详细说明。该文档包含了游戏中所有道具相关的日志类型定义。

## 日志类型区间定义

### 1. 玩家获得物品 (ILG - Item Log Gain)
- **区间**: `60001 - 69999`
- **常量**: `ILG_MIN = 60000`, `ILG_MAX = 69999`
- **记录场景**: 玩家获得道具的所有渠道
- **记录数量**: `item_num > 0`

### 2. 玩家失去物品 (ILL - Item Log Loss)
- **区间**: `70001 - 79999`
- **常量**: `ILL_MIN = 70000`, `ILL_MAX = 80000`
- **记录场景**: 玩家消耗、丢弃、交易失去道具的所有渠道
- **记录数量**: `item_num < 0`

### 3. 玩家移动物品 (ILI - Item Log Internal)
- **区间**: `80001 - 89999`
- **常量**: `ILI_MIN = 80000`, `ILT_MAX = 90000`
- **记录场景**: 玩家内部道具移动，如仓库转移、背包整理等

## 分析应用建议

### 1. 经济系统监控
- **获得/消耗平衡**: 监控 60000+ 与 70000+ 系列的比例
- **渠道健康度**: 分析各渠道的产出/消耗合理性
- **异常检测**: 识别异常渠道活动

### 2. 玩家行为分析
- **获取偏好**: 分析玩家偏好哪些获得渠道
- **消耗模式**: 研究玩家的道具消耗习惯
- **行为异常**: 检测异常获取或消耗行为

### 3. 系统优化参考
- **渠道效率**: 评估各渠道的参与度和产出效率
- **用户体验**: 分析玩家在不同渠道的体验
- **平衡调整**: 为游戏平衡调整提供数据支持

## 数据查询示例

### 查询特定渠道数据
```sql
-- 查询60001渠道的获得记录
SELECT * FROM item_table 
WHERE logtype = '60001' 
  AND item_num > 0
  AND substr(time, 1, 10) = '2026-03-25';

-- 查询70001渠道的消耗记录
SELECT * FROM item_table 
WHERE logtype = '70001' 
  AND item_num < 0
  AND substr(time, 1, 10) = '2026-03-25';
```

### 渠道对比分析
```sql
-- 对比核心产出渠道
SELECT logtype, COUNT(*) as count, SUM(item_num) as total
FROM item_table
WHERE logtype IN ('60001', '60008', '60012', '60015', '61108', '61160')
  AND item_num > 0
  AND substr(time, 1, 10) = '2026-03-25'
GROUP BY logtype
ORDER BY total DESC;
```
